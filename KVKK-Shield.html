Active R&D Project

KVKK Shield: Client-Side Data Guardrail

Solving the absolute biggest compliance friction point for Turkish enterprises using public Generative AI.

The Context & Workplace Pain

Under the Turkish Personal Data Protection Law (KVKK), copying and pasting customer data or personal identifiers (T.C. IDs, local phone numbers, emails) into public AI models (like ChatGPT or Claude) is classified as an illegal cross-border data transfer. Following the **March 12, 2026 KVKK directives**, companies face severe audit liabilities and fines for "BYOAgentic" workflows used by employees without technical safeguards. Total bans are impractical and hurt workplace productivity.

The Innovation

**KVKK Shield** is a browser extension running 100% client-side. It intercepts chat penders instantly via DOM Event Listeners. Before any text payload is compiled and sent to US-based servers, our local regex algorithms sanitize Turkish PII patterns (TCKN, local phone numbers, emails) and replace them with secure, generalized placeholders. Real data never leaves the local machine.

Active Extension Prototype (content.js)

This is the actual, lightweight content script running client-side inside ChatGPT/Claude to sanitize Turkish personal data in real-time. 

// Turkish T.C. ID Number RegEx Pattern
const tcknRegex = /\b[1-9]\d{10}\b/g;

// Real-time sanitization function
function maskSensitiveData(text) {
  return text.replace(tcknRegex, "<REDACTED_TCKN>");
}

// Intercept Prompt DOM submission
document.addEventListener('keydown', (event) => {
  const target = event.target;
  if (target && target.tagName === 'TEXTAREA') {
    if (event.key === 'Enter') {
      target.value = maskSensitiveData(target.value);
    }
  }
}, true);
